## 使用配置文件以及文件描述

- 什么是文件描述符？
  首先，系统内核会为所有进程维护一张当前打开的所有文件和资源的表格。
  每个打开的文件都会分配一个名为“文件描述符”的数值标识。
  在系统层，所有的文件系统操作，都会使用这些文件标识符来指定特定的文件。
  在 window 下，会使用不同但概念类似的机制来追踪文件和资源。
  node 抽象了不同系统之间的差异，为所有打开的文件分配和数值的文件描述符。
